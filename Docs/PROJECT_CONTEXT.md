# 📄 سياق المشروع (Project Context)

## 🌟 فكرة المشروع وأهدافه
مشروع **طبيبي (Tabibi)** هو منصة طبية شاملة تستهدف السوق الجزائري، تهدف إلى تسهيل إدارة العيادات الطبية، تنظيم مواعيد المرضى، وربط الأطباء بالمرضى بشكل مباشر. يخدم المشروع عدة أطراف: المريض، الطبيب، العيادة، ومسؤول النظام (الأدمن). بالإضافة إلى ذلك، يوفر النظام تكاملاً فريداً (Synchronization) مع نظام مكتبي (Desktop) مبني بلغة Delphi يستخدم محلياً في العيادات.

## ✨ المميزات الحالية
- **إدارة المستخدمين والصلاحيات**: نظام مصادقة يعتمد على Tokens ويدعم أدوار متعددة (مريض، طبيب، عيادة، أدمن).
- **البحث المتقدم والفلترة**: يمكن للمريض البحث عن العيادات والأطباء حسب التخصص والولاية/البلدية.
- **إدارة المواعيد**: نظام ذكي يعرض الأوقات المتاحة بناءً على جدول عمل الطبيب، ويسمح بالحجز المباشر وإلغاء المواعيد.
- **التواصل الفوري**: نظام دردشة مدمج بين المريض والطبيب.
- **التقييمات**: إمكانية تقييم المرضى للأطباء بعد الانتهاء من الموعد.
- **نظام التذاكر (الدعم الفني)**: لتقديم الدعم وحل المشاكل التقنية للمستخدمين.
- **المزامنة (Synchronization)**: نظام مخصص لمزامنة البيانات بين الخادم السحابي (API) وتطبيق العيادة المكتبي (Delphi) لضمان عدم حدوث تعارض في المواعيد بين الحجز الأونلاين والحجز المحلي.

## 🏗️ البنية العامة للمشروع
يعتمد المشروع على بنية مفصولة بالكامل (Decoupled Architecture):
1. **Frontend (الواجهة الأمامية)**: 
   - مبني باستخدام **React.js 18** و **Vite**.
   - تطبيق صفحة واحدة (SPA) يستخدم Hash Routing.
2. **Backend (الخادم وواجهة برمجة التطبيقات)**:
   - مبني بلغة **PHP (Vanilla / بدون إطار عمل)**.
   - يعتمد على بنية تشبه MVC مصغرة لتنظيم الطلبات.
3. **Database (قاعدة البيانات)**:
   - **MySQL / MariaDB** بعلاقات معقدة ومفاتيح أساسية من نوع UUID.
4. **Mobile (تطبيق الجوال)**:
   - تطبيق هجين باستخدام **Capacitor** لتغليف تطبيق React ويعمل على نظام أندرويد.
5. **Desktop (تطبيق سطح المكتب)**:
   - تطبيق Delphi يتواصل مع الـ Backend لتبادل البيانات (نظام غير متصل بشكل دائم - يعمل Offline ويتزامن عند الحاجة).

## 🔄 طريقة تدفق البيانات داخل النظام
1. **المصادقة**: يسجل المستخدم دخوله، فيقوم الخادم بالتحقق وإرجاع `Bearer Token` يُحفظ في الـ `LocalStorage` للمتصفح.
2. **الاستعلام**: يرسل الـ Frontend طلبات HTTP إلى `/api/*` مع إرفاق الـ Token في ترويسة (Header) الطلب.
3. **المعالجة**: يستقبل ملف `index.php` (الـ Router) الطلب، ويمرره عبر الـ `AuthMiddleware` للتحقق من الصلاحيات، ثم يحوله للـ `Controller` المناسب.
4. **قاعدة البيانات**: يستخدم الـ Controller كلاس `Database` (وهو Singleton PDO) لجلب أو تحديث البيانات بأمان.
5. **الاستجابة**: يعيد الخادم البيانات بتنسيق JSON موحد عبر كلاس `Response`.
6. **المزامنة**: يقوم تطبيق Delphi بطلب `/api/sync/*` بشكل دوري لرفع المواعيد الجديدة المسجلة محلياً وجلب المواعيد المحجوزة أونلاين.

## 🚀 طريقة التشغيل والإعداد المحلي (مختصر)
1. تشغيل سيرفر محلي (Apache/MySQL) عبر XAMPP أو WAMP أو غيره.
2. استيراد قاعدة البيانات المرفقة للمشروع (`uyyuppcc_DBTabibi_1_.sql`).
3. إعداد الاتصال بقاعدة البيانات والبريد الإلكتروني في ملف الإعدادات `backend/config/database.php`.
4. تشغيل الخادم المحلي للـ Backend (مثلاً: `php -S localhost:8000 -t backend`).
5. تثبيت حزم الـ Frontend بالدخول للمجلد وتشغيل `npm install` ثم بدء خادم التطوير `npm run dev`.
> **ملاحظة**: لمزيد من التفاصيل الدقيقة حول خطوات التشغيل ومتطلبات البيئة، راجع ملف `SETUP_GUIDE.md`.
