# ✅ قائمة المهام والتحسينات (Tasks & Improvements)

هذا الملف يحتوي على قائمة المهام المفتوحة (To-Do)، المشاكل التقنية المكتشفة، واقتراحات التطوير والتحسين لمشروع طبيبي. يمكن لأي مطور أو مساعد ذكاء اصطناعي الرجوع لهذه القائمة كخارطة طريق (Roadmap).

## 🛠️ 1. المشاكل التقنية والديون التقنية (Technical Debt)
تم كشف بعض النقاط المعمارية والبرمجية في الكود الحالي التي يجب العمل على تحسينها لاحقاً:

- **مشكلة حجم `App.jsx`**: ملف التوجيه والواجهة الرئيسي في الـ React حجمه ضخم جداً (حوالي 246KB). هذا يجعل صيانته صعبة.
  - **الحل المقترح**: تقسيم الملف إلى مكونات أصغر (Component Refactoring)، واستخدام مكتبة موحدة ومستقرة مثل `react-router-dom` بدلاً من الـ Hash Routing اليدوي المبني بملف `useRoute.js`.
- **مشكلة التشفير الضعيف للمرور (Base64 Password)**: النظام حالياً يستخدم Base64 لتشفير كلمات المرور. هذا تشفير غير آمن إطلاقاً ويُعتبر Encoding وليس Hashing. السبب في ذلك هو التوافق مع تطبيق Delphi.
  - **الحل المقترح**: تحديث تطبيق Delphi لدعم خوارزميات الـ Hash الآمنة (مثل `Bcrypt` أو `Argon2`)، ثم تحديث كود الـ PHP (عبر `password_hash` و `password_verify`).
- **تكرار منطق الاستعلامات (SQL Duplication)**: بعض الـ Controllers تقوم بكتابة استعلامات SQL طويلة بشكل متكرر.
  - **الحل المقترح**: إعداد طبقة (Repository Pattern) أو استخدام Object-Relational Mapping (ORM) بسيط لتجميع الاستعلامات المشتركة.

## 🚀 2. التطويرات المستقبلية المقترحة (Future Enhancements)
- **إشعارات الوقت الفعلي (Real-time Notifications)**: دمج نظام (WebSockets أو Pusher) لإرسال الإشعارات للمرضى فور قبول المواعيد من العيادة أو في حال تلقي رسالة جديدة في الدردشة بدلاً من عمل Refresh للصفحة.
- **تطوير تطبيق الهاتف**: إنهاء اختبارات تطبيق الـ Capacitor وإضافة دعم الـ Push Notifications للموبايل.
- **ترقية الـ Backend إلى إطار عمل (Framework Migration)**: بدلاً من استخدام PHP الصريح، قد يكون من الجيد التفكير بنقل الـ API إلى **Laravel** أو **NestJS** مستقبلاً للحصول على أمان، استقرار، وسرعة في إضافة الميزات.
- **الدفع الإلكتروني (E-Payment)**: إضافة بوابات دفع (مثل CIB أو البطاقة الذهبية في الجزائر) لدفع رسوم الحجز المسبق لبعض العيادات لتقليل نسب عدم الحضور (No-shows).
